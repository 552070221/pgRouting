Dijkstra from storage to process & from process to result

The data transformation

The data for the process is stored in the postgreSQL database, and it is requiered to insert it into the algorithm structures. The \( edges_sql \) is interpreted in the following way:

The Query

Given the following query:

  • pgr_dijkstra( \( edges_sql, start_{vid}, end_{vid}, directed \))

    where: The \( edges_sql \) is a query that returns a set of stored \( Edges \): \( Edges = \{(id_i, source_i, target_i, cost_i, reverse\_cost_i)\} \)

Let:

  • \( source = \bigcup source_i \)
  • \( target = \bigcup target_i \)

The set of vertices \( V \) is defined as follows

\( V = source \cup target \cup {start_{vid}} \cup {end_{vid}} \)

In other words, it includes:

  • All the vertices in the \( {source_{vid}} \) column
  • All the vertices in the \( {target_{vid}} \) column
  • The \( {start_{vid}} \) parameter
  • The \( {end_{vid}} \) parameter

Weighted Directed Graph

The weighted undirected graph, \( G_d(V,E) \) defined by:

  • The set of vertices \( V \)

  • The set of edges \( E \) defined by:

    \( E = \begin{cases} \{ (source_i, target_i, cost_i) \text{ when } cost_i >=0 \} &\quad \text{ if } reverse\_cost = \varnothing \\ \\ \text{-------- } \\ \begin{array} \\ \{ (source_i, target_i, cost_i) \text{ when } cost_i >= 0 \} \\ \cup \\ \{ (target_i, source_i, reverse\_cost_i) \text{ when } reverse\_cost_i >=0 \} \end{array} &\quad \text{ if } reverse\_cost \neq \varnothing \end{cases} \)

Weighted Undirected Graph

The weighted undirected graph, \( G_u(V,E) \) defined by:

  • The set of vertices \( V \)

  • The set of edges \( E \) defined by:

    \( E = \begin{cases} \begin{array} \\ \{ (source_i, target_i, cost_i) \text{ when } cost_i >=0 \} \\ \cup \\ \{ (target_i, source_i, cost_i) \text{ when } cost_i >=0 \} \end{array} &\quad \text{ if } reverse\_cost = \varnothing \\ \\ \text{-------- } \\ \begin{array} \\ \{ (source_i, target_i, cost_i) \text{ when } cost_i >=0 \} \\ \cup \\ \{ (target_i, source_i, cost_i) \text{ when } cost_i >=0 \} \\ \cup \\ \{ (source_i, target_i, reverse\_cost_i) \text{ when } reverse\_cost_i >=0)\} \\ \cup \\ \{ (target_i, source_i, reverse\_cost_i) \text{ when } reverse\_cost_i >=0)\} \end{array} &\quad \text{ if } reverse\_cost \neq \varnothing \\ \end{cases} \)

Solving Dijkstra

Given:

  • \( G(V,E) = \begin{cases} G_d(V,E) &\quad \text{ if } directed = true \\ G_u(V,E) &\quad \text{ if } directed = false \\ \end{cases} \)
  • \( start_{vid} \in V \) a starting vertex
  • \( end_{vid} \in V \) an ending vertex

Then: \( \text{pgr_dijkstra}(sql, start_{vid}, end_{vid}, directed) = \begin{cases} \boldsymbol{\pi} &\quad \text{if } \exists \boldsymbol{\pi} \\ \varnothing &\quad \text{otherwise} \\ \end{cases} \)

\( \boldsymbol{\pi} = \{(path\_seq_j, node_j, edge_j, cost_j, agg\_cost_j) \} \)

where:

  • \( path\_seq_j = j \)
  • \( path\_seq_{| \pi |} = | \pi | \)
  • \( node_j \in V \)
  • \( node_1 = start_{vid} \)
  • \( node_{| \pi |} = end_{vid} \)
  • \( \forall j \neq | \pi |, \quad (node_j, node_{j+1}, cost_j) \in E \)
  • \( edge_j = \begin{cases} id_{(node_j, node_{j+1},cost_j)} &\quad \text{when } j \neq | \pi | \\ -1 &\quad \text{when } j = | \pi | \\ \end{cases} \)
  • \( cost_j = cost_{(node_j, node_{j+1})} \)
  • \( agg\_cost_j = \begin{cases} 0 &\quad \text{when } j = 1 \\ \displaystyle\sum_{k=1}^{j} cost_{(node_{k-1}, node_k)} &\quad \text{when } j \neq 1 \\ \end{cases} \)

In other words:

  • The algorithm returns a the shortest path between \( start_{vid} \) and \( end_{vid} \), if it exists, in terms of a sequence of nodes and of edges,
    • \( path\_seq \) indicates the relative position in the path of the \( node \) or \( edge \)
    • \( cost \) is the cost of the edge to be used to go to the next node.
    • \( agg\_cost \) is the cost from the \( start_{vid} \) up to the node.
  • If there is no path, the resulting set is empty.
