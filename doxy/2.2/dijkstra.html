Name

pgr_dijkstra - a family of functions that perform dijkstra algorithm

Synopsis

Minimal signature

This signature performs the Dijkstra algorithm from start_vid to end_vid 

pgr_dijkstra(edges_sql, start_vid, end_vid)
RETURNS SET OF (seq, path_seq, node, edge, cost, agg_cost) or EMPTY SET

Example:

SELECT * FROM pgr_dijkstra(
'SELECT id, source, target, cost, reverse_cost FROM edge_table',
2, 3
);

Results:

seq | path_seq | node | edge | cost | agg_cost
-----+----------+------+------+------+----------
1 | 1 | 2 | 4 | 1 | 0
2 | 2 | 5 | 8 | 1 | 1
3 | 3 | 6 | 9 | 1 | 2
4 | 4 | 9 | 16 | 1 | 3
5 | 5 | 4 | 3 | 1 | 4
6 | 6 | 3 | -1 | 0 | 5
(6 rows)

One to One

This signature performs the Dijkstra algorithm from start_vid to end_vid: 

pgr_dijkstra(edges_sql, start_vid, end_vid, directed:=true)
RETURNS SET OF (seq, path_seq, node, edge, cost, agg_cost) or EMPTY SET

Example:

SELECT * FROM pgr_dijkstra(
'SELECT id, source, target, cost, reverse_cost FROM edge_table',
2, 3,
false
);

Results:

seq | path_seq | node | edge | cost | agg_cost
-----+----------+------+------+------+----------
1 | 1 | 2 | 2 | 1 | 0
2 | 2 | 3 | -1 | 0 | 1
(2 rows)

Parameters

Types and description of the types.

Column Type Description
edges_sqlTEXT SQL query as decribed bellow.
start_vidBIGINTIdentifier of the starting vertex of the path.
end_vidBIGINTIdentifier of the ending vertex of the path.
directedBOOLEANDefault is true (is directed). When set to false the graph is considered as Undirected.
end_vidsARRAY[ANY-INTEGER]Array of identifiers of ending vertices.
start_vidsARRAY[ANY-INTEGER]Array of identifiers of starting vertices.

edges_sql is an SQL query, which returns a set of rows with the following columns:

Column Type Description
id ANY-INTEGER Identifier of the edge.
sourceANY-INTEGERIdentifier of the first end point vertex of the edge.
targetANY-INTEGERIdentifier of the second end pont vertex of the edge.
costANY-NUMERICALWeight of edge (source, target)
- When negative: the edge (source, target) is not inserted
- When positive: the edge (source, target) is inserted,
- If graph is undirected: the rules also applu tp the edge (target, source)
reverse_costANY-NUMERICALWeight of edge (target, source)
- When negative: the edge (target, source) is not inserted
- When positive: the edge (target, source) is inserted,
- If graph is undirected: the rules also apply tp the edge (source, target)

Return values

Depending of the signature it will return the following columns 

RESTURNS SET OF (seq, path_seq [, start_vid] [, end_vid], node, edge, cost, agg_cost)
Column Type Description
seqINT Sequential value starting from 1.
path_seqINT Relative position in the path. Has value 1 for the begining of a path.
start_vidBIGINT Identifier of the starting vertex.
Used when multiple starting veritices are in the query.
end_vidBIGINT Identifier of the ending vertex.
Used when multiple ending vertices are in the query.
nodeBIGINT Identifier of the node in the path from start_vid to end_vid.
edgeBIGINT Identifier of the edge used to go from node to the next node in the path sequence.
**-1** for the last node of the path.
costFLOAT Cost to traverse from node using edge to the next node in the path sequence.
agg_costFLOAT Total cost from start_vid to node.
